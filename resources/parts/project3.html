Per implementare una WebApp si può usare Spring Boot. Tuttavia, ci sarebbero dei problemi: ogni volta per aggiornare la pagina web, dato che lo stato può cambiare, il manager dovrebbe fare una richiesta di polling. Sicuramente, questa soluzione è inefficiente. Per cui per superare questo problema si usano anche le socket.io La pagina web viene caricata solo una volta durante la richiesta HTTP request mentre il contenuto al suo interno può cambiare non dovendo ricaricare la pagina.
